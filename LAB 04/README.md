# 💻 CSC-521 Object Oriented Programming – Lab 04

This lab focuses on understanding and implementing **constructors in C++**. Through three practical tasks, students will explore default constructors, parameterized constructors, and constructor overloading.

---

# ⭐ Task 1: Default Constructor – Employee Management System
## 🔍 Overview

This program demonstrates how to use a **default constructor** to automatically initialize employee details when an object is created.

## 🎯 Objective

To understand how default constructors provide initial values and ensure objects start in a valid state.

## ✔️ Output

Displays employee details with default values before any updates.

## ✍️ Learning Outcomes

- Define and use default constructors
- Automatically initialize object data
- Improve code safety and readability

---

# ⭐ Task 2: Parameterized Constructor – Bank Account Initialization
## 🔍 Overview

This program uses a **parameterized constructor** to initialize account details such as account number, account holder, and balance at the time of object creation.

## 🎯 Objective

To practice initializing objects with custom values provided at runtime.

## ✔️ Output

Displays account details initialized through the parameterized constructor.

## ✍️ Learning Outcomes

- Create parameterized constructors
- Pass arguments during object creation
- Manage object data efficiently

---

# ⭐ Task 3: Constructor Overloading – Rectangle Example
## 🔍 Overview

This program demonstrates **constructor overloading** by defining multiple constructors for a `Rectangle` class. It includes a default constructor, a parameterized constructor, and a single-parameter constructor for squares.

## 🎯 Objective

To understand how multiple constructors can provide flexibility in object creation.

## ✔️ Output

Displays rectangle dimensions and area using different constructor versions.

## ✍️ Learning Outcomes

- Implement constructor overloading
- Differentiate between default, parameterized, and single-parameter constructors
- Apply constructors to real-world examples

---

# ⭐ Task 4: Destructor – Locker System
## 🔍 Overview

This task involves creating a `Locker` class that prints messages when allocated and returned. The destructor is automatically called when the object goes out of scope or is deleted.

## 🎯 Objective

To understand destructors and how they manage cleanup when objects are destroyed.

## ✔️ Output

Displays messages showing when a locker is allocated and when it is returned.

## ✍️ Learning Outcomes

- Implement destructors in C++  
- Understand automatic cleanup of resources  
- Differentiate between stack and heap object destruction
